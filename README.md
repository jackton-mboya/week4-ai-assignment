## Question 1: What is Data?
Data is a collection of raw facts, figures, or observations.

**Examples:**
1. A list of students' names in a classroom.
2. The number of steps recorded by a fitness tracker in a day.

## Question 2: Structured vs. Unstructured Data
The main difference is how the data is organized.

* **Structured Data:** Highly organized and fits into fixed fields (like a spreadsheet).
    * *Examples:* A bank statement with dates and amounts; A contact list with Names and Phone Numbers.
* **Unstructured Data:** Has no pre-defined format, making it harder to search.
    * *Examples:* A collection of voice notes; A PDF of a handwritten essay.



## Question 3: What is a Schema?
A schema is a "blueprint" or a set of rules that defines how data is organized and what type of information is allowed in a specific place.

**Example:** Think of a **Restaurant Menu**. The schema dictates that there is a "Section Name" (Appetizers), an "Item Name" (Spring Rolls), and a "Price" ($5.00). It ensures the price isn't written where the name should be.

## Bonus Question (Optional)
Explain the difference between:
A list
A dictionary

You access items in a list by their position (index), while in a dictionary, you access items by a unique label (key).
A list is a simple sequence of values, whereas a dictionary is a collection of key-value pairs (data connected to a name).
You use a list when the order of items matters (like a ranking), but you use a dictionary when you want to label your data for easy lookup.
